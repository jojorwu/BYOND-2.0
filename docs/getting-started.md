# Начало работы с BYOND 2.0

Это руководство поможет вам быстро начать работу с игровым движком BYOND 2.0.

## 1. Предварительные требования

Для работы с проектом вам понадобится **.NET 8.0 SDK**.

### Установка .NET 8.0 SDK

Вы можете установить .NET 8.0 SDK, выполнив следующие команды в корневой папке проекта:

```bash
chmod +x ./dotnet-install.sh
./dotnet-install.sh --channel 8.0
```

После установки необходимо добавить .NET в `PATH` для текущей сессии:

```bash
export DOTNET_ROOT=$HOME/.dotnet
export PATH=$PATH:$DOTNET_ROOT:$DOTNET_ROOT/tools
```

## 2. Структура проекта

*   `Core/`: Общая библиотека с основной логикой движка.
*   `Server/`: Серверное приложение.
*   `Editor/`: Графический редактор для создания игровых миров.
*   `scripts/`: Каталог со скриптами на Lua, определяющими логику вашей игры.
*   `assets/`: Ресурсы, используемые в проекте (графика, звуки).
*   `project.json`: Файл настроек вашего проекта.

## 3. Запуск Редактора

Редактор — это основной инструмент для создания и редактирования игровых миров.

> **Примечание:** В текущей среде выполнения запуск графического редактора невозможен из-за отсутствия необходимых библиотек. Эти инструкции предназначены для пользователей, работающих в полноценной среде рабочего стола.

Чтобы запустить редактор, выполните команду:

```bash
dotnet run --project Editor/Editor.csproj
```

При первом запуске редактор предложит вам создать новый проект. После создания проекта в его корневой папке будет создан `Server.exe`.

## 4. Запуск Сервера

Для запуска игрового сервера просто запустите `Server.exe` в папке вашего проекта.

Сервер автоматически загрузит карту, указанную в `MainMap` в файле `project.json`, а затем выполнит скрипты из каталога `scripts`.

## 5. Что дальше?

*   **Архитектура:** Изучите `architecture.md`, чтобы глубже понять, как устроен движок.
*   **Скриптинг:** Ознакомьтесь с `scripting.md`, чтобы узнать, как использовать API `Game` для создания игровой логики.
